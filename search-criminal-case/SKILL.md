# 计算机犯罪类案检索 Skill

面向律师等法律从业者的计算机犯罪类案检索与量刑差异分析工作流。将"类案检索四步法"与"要件对比分析"两套方法论编排为六阶段工作流，内置**要素自动提取引擎**与**检索策略生成引擎**，集成北大法宝双 MCP（语义检索 + 关键词检索）与本地案例库，自主完成从案情输入到类案检索报告与个案裁判对比分析的全流程。

裁判依据：《中华人民共和国刑法》（2023 修正）及现行司法解释。聚焦数据犯罪四个特定罪名的"定性困难"现象，以量化对比与裁判逻辑分析辅助办案决策。

---

## 一、快速开始

### 1.1 适用场景

用户提交计算机犯罪案件的关键情节要素，并希望快速获得类案量刑对比分析报告时——典型场景：

- **辩护律师**为计算机犯罪案件做案件罪名定性，调用本地/在线数据库进行案例检索，根据检索结果通过预设的分组规则进行定量分析，为辩护策略提供数据支撑；
- **法律研究者**对计算机犯罪四个特定罪名的裁判结果做实证分析；

**聚焦罪名**：

| 序号 | 罪名 | 法条 |
|------|------|------|
| 1 | 非法侵入计算机信息系统罪 | 《刑法》第285条第1款 |
| 2 | 非法获取计算机信息系统数据、非法控制计算机信息系统罪 | 《刑法》第285条第2款 |
| 3 | 提供侵入、非法控制计算机信息系统程序、工具罪 | 《刑法》第285条第3款 |
| 4 | 破坏计算机信息系统罪 | 《刑法》第286条 |

**不适用**：其他罪名的计算机犯罪类案检索、起诉书/辩护词等诉讼文书撰写（请使用专门的文书 skill）、不涉及上述四罪名的案件分析。

### 1.2 前置依赖

| 依赖项 | 说明 |
|--------|------|
| 北大法宝 MCP（语义检索） | 推荐，语义类案检索 |
| 北大法宝 MCP（关键词检索） | 推荐，精准关键词检索 |
| 本地案例库 | 可选，离线案例检索 |
| Python 3.12+ | 必需 |
| python-docx | Word文档生成依赖 |

### 1.3 六阶段工作流概览

```
Phase 1          Phase 2          Phase 2.5            Phase 3              Phase 4          Phase 5          Phase 6
案情解析    →    罪名发现    →    检索模式选择    →    预选确认         →    大纲确认    →    并行写作    →    案例入库
(要素提取引擎)    (要件对比分析)    (决策节点)           + 类案检索            (报告编排)       + Word 输出       (决策节点)  
  (检索策略生成引擎)
```

| 阶段 | 核心动作 | 用户交互 | 状态 |
|------|----------|----------|------|
| 1 案情解析 | 要素自动提取 + 一键修正引导 | ⛔ 修正要素 + ⛔ 保存反馈 | ✅ 已实现 |
| 2 罪名发现 | 要件对比分析+竞合判断 | ⛔ 手动选择罪名（自动识别失败时） | ✅ 已实现 |
| 2.5 检索模式选择 | 判断可用数据源 | ⛔ 选择检索模式 | ⚠️ 当前默认本地模式 |
| 3 预选确认+检索 | 确认→策略生成→检索→统计 | ⛔ 确认检索 | ✅ 已实现 |
| 4 大纲确认 | 报告大纲生成 | ⛔ 确认大纲和类案 | ⏳ 待实现（当前直接生成） |
| 5 并行写作+输出 | 分节写作→合并→Word生成 | 无 | ⏳ 待实现并行（当前串行） |
| 6 案例入库决策 | 展示新案例→用户选择→入库 | ⛔ 选择入库方式 | ⏳ 待实现 |

## 二、各阶段详细说明

### 2.1 Phase 1 — 案情解析（要素提取引擎）

**输入**：用户提交的案情叙述

**输出**：结构化案情要素表

**核心动作**：从案情叙述中自动提取以下要素（缺失字段填"未提及"，严禁臆测）：

| 要素名称 | 提取内容 | 可选值 |
|----------|----------|--------|
| 案情概述 | 一句话案情概述 | - |
| 行为类型 | 侵入/获取数据/非法控制/提供程序工具/破坏系统/组合行为 | 侵入、获取数据、非法控制、提供程序工具、破坏系统、组合行为 |
| 目标系统类型 | 系统类型 | 国家事务、国防建设、尖端科技、其他重要领域、普通系统 |
| 系统具体描述 | 如某省政府门户网站、某银行系统等 | - |
| 技术手段 | SQL注入、暴力破解、木马植入等 | SQL注入、暴力破解、木马植入、钓鱼、社工、数据爬取 |
| 数据类型 | 个人信息/账号密码/金融数据/系统控制权/程序工具/不涉及 | 个人信息、账号密码、金融数据、系统控制权、程序工具、不涉及 |
| 数据量 | 条数/金额/数量 | - |
| 后果 | 系统瘫痪/数据泄露/经济损失/未造成严重后果/其他 | 系统瘫痪、数据泄露、经济损失、未造成严重后果、其他 |
| 后果严重程度 | 严重/特别严重/未提及 | 严重、特别严重、未提及 |
| 经济损失 | 具体经济损失金额 | - |
| 被告人角色 | 实行者/组织者/帮助者/提供者 | 实行者、组织者、帮助者、提供者 |
| 主观故意 | 直接故意/间接故意/过失 | 直接故意、间接故意、过失 |
| 特殊情节 | 如未成年人/累犯/共同犯罪/退赔退缴等 | - |

**输出格式**：Markdown表格 + 要素缺失提示（⚠️ 高亮未提及字段）

**一键修正引导**：
1. 高亮显示所有"未提及"字段，附带补充建议和可选值列表
2. 支持按序号选择字段进行交互式修正
3. 修正后可标记"提取错误"，反馈数据保存至 `input/training_data/` 用于模型迭代

**用户决策环节**：
- **⛔ 是否修正要素**：系统询问用户是否需要修正未提及或错误的要素
- **⛔ 交互式修正**：用户可按序号选择字段，从预设选项中选择或直接输入自定义值
- **⛔ 是否保存反馈**：用户可选择将修正数据保存为训练数据，用于后续模型迭代

### 2.2 Phase 2 — 罪名发现（要件对比分析）

**输入**：案情要素表

**输出**：候选罪名列表 + 竞合关系判断

**核心动作**：

1. **要件对比**：将案情要素与四罪名要件进行逐条比对
2. **罪名匹配**：根据行为类型和系统类型匹配候选罪名
3. **竞合判断**：分析罪名间的牵连/吸收/数罪并罚关系

**输出格式**：
- 候选罪名列表（含匹配度）
- 要件对比分析表
- 竞合关系说明

**用户决策环节**：
- **⛔ 手动选择罪名**：当自动识别失败或未识别到罪名时，系统展示可选罪名列表，用户需按序号选择

### 2.3 Phase 2.5 — 检索模式选择（决策节点）

**输入**：北大法宝MCP连接状态

**输出**：检索模式选择

**决策逻辑**：

```
检查北大法宝 MCP 是否可用？
    │
    ├─ 是 → 在线+本地合并模式（推荐）
    │       数据源：MCP语义检索 + 本地案例库
    │
    └─ 否 → 纯本地模式
            数据源：仅本地案例库
            提示：数据量可能不足
```

**用户交互**：展示MCP状态和两种模式对比，等待用户确认

**当前状态**：默认使用纯本地模式（本地案例库34个案例），MCP接入后可切换至在线+本地合并模式

### 2.4 Phase 3 — 预选确认与类案检索

**输入**：候选罪名 + 案情要素

**输出**：类案结果集 + 统计分析

**核心动作**：

1. **预选确认**（⛔ 用户确认）：确认罪名选择、补充要素
2. **策略生成**：根据罪名和要素生成语义检索策略
3. **检索执行**：调用MCP（如选择在线模式）+ 本地库检索
4. **合并去重**：按"案号+罪名"去重，排除非实体判决
5. **分组统计**：按用户确认的罪名过滤，统计量刑分布等

**关键规则**：统计分析必须按用户确认的罪名进行过滤

### 2.5 Phase 4 — 大纲确认（报告编排）

**输入**：检索结果 + 统计分析

**输出**：报告大纲 + 典型类案选定

**核心动作**：

1. **生成大纲**：基于检索结果生成报告大纲
2. **选定典型类案**：选取5件典型案例（最轻/最重/中位数/指导案例/最相似）
3. **用户确认**（⛔）：展示大纲和类案列表，等待确认

**当前状态**：⏳ 待实现（当前版本直接生成报告，跳过大纲确认环节）

### 2.6 Phase 5 — 并行写作与 Word 输出

**输入**：报告大纲

**输出**：Word文档 + Markdown备份

**核心动作**：

1. **并行写作**：分章节并行写作（案件概述+罪名分析、检索说明+统计分析、典型类案评析、启示）
2. **报告生成**：合并各章节，生成格式化报告
3. **Word输出**：使用python-docx生成Word文档

**报告命名**：`检索报告-[YYYY-MM-DD]-[HHMM].docx`（首次）/ `检索报告-[YYYY-MM-DD]-[序号]-[HHMM].docx`（同一天第二次及以后）  
**保留机制**：同一天多次检索自动递增序号 + 时分戳，保留所有历史报告，避免覆盖  

**临时文件管理**：
- 所有中间文件写入 `output/temp/` 目录
- 报告生成成功后自动清理临时文件
- 使用 `try...finally` 确保异常中断后也能清理
- 启动时自动清理残留临时文件

**当前状态**：⏳ 待实现并行写作（当前版本为串行模式），临时文件管理已实现

### 2.7 Phase 6 — 案例入库决策（决策节点）

**输入**：MCP检索到的新案例

**输出**：入库结果

**核心动作**：

1. **展示新案例**：列出MCP检索到的未入库案例
2. **用户选择**（⛔）：全部入库/逐条选择/不入库 + 入库方式（仅JSON/JSON+txt）
3. **执行入库**：调用save_to_local.py保存到本地数据库
4. **验证去重**：确认入库成功且无重复

**当前状态**：⏳ 待实现（save_to_local.py已实现，但主流程中未调用）

### 3.3 罪名分析详细论证要求

每个罪名分析必须包含：

1. **构成要件分析**：
   - 客体要件：明确本罪侵犯的客体
   - 客观要件：明确客观方面表现
   - 主体要件：明确主体范围
   - 主观要件：明确主观方面

2. **本案符合性论证**：
   - 违反国家规定：引用具体法律条文（如《网络安全法》第二十七条）
   - 行为要件：论证本案事实符合法律规定的行为要件
   - 情节严重/特别严重：引用司法解释（法释〔2011〕19号）的量化标准
   - 主观故意：分析被告人的主观心态

3. **竞合关系辨析**：
   - 牵连关系论证：分析手段行为与目的行为的关系
   - 择一重罪论处：比较两罪的法定刑，说明择一重罪的理由

### 3.4 案例多维相似度比对要求

典型类案评析必须包含：

1. **多维相似度比对表**（6维度+权重评分）：

| 对比维度 | 权重 | 案例1 | 案例2 | 案例3 | 案例4 | 案例5 | 本案 |
|----------|------|-------|-------|-------|-------|-------|------|
| 手段 | 25% | ... | ... | ... | ... | ... | ... |
| 数据来源 | 20% | ... | ... | ... | ... | ... | ... |
| 行为人身份 | 20% | ... | ... | ... | ... | ... | ... |
| 数据量 | 15% | ... | ... | ... | ... | ... | ... |
| 获利金额 | 10% | ... | ... | ... | ... | ... | ... |
| 跨境情节 | 10% | ... | ... | ... | ... | ... | ... |
| 综合相似度 | 100% | [评分] | [评分] | [评分] | [评分] | [评分] | - |

2. **参考价值说明**：
   - 相同点：与本案相同的核心要素
   - 不同点：与本案不同的核心要素
   - 参照价值：星级评定（★★★★★-★☆☆☆☆）

3. **类案局限性说明**：分析本案核心特征在现有类案中缺乏直接对应的方面

**评分规则**：完全相同100分，高度相似80-99分，中等相似50-79分，低度相似20-49分，几乎不相似0-19分

**星级评定**：5星（≥80分）、4星（60-79分）、3星（40-59分）、2星（20-39分）、1星（<20分）

## 四、操作规范

### 4.1 检索策略规范

- **上限控制**：每个罪名仅获取3条最相似案例，控制token消耗
- **本地优先**：当本地案例库足够丰富时，可完全跳过MCP调用
- **按需检索**：仅在用户确认调用MCP时才执行在线检索
- **排除列表**：调用MCP时必须传递 `exclude_case_numbers` 参数，排除本地数据库已入库的案例
- **参数配置**：MCP调用参数由 `config/default.json` 管理，包括检索模式、返回数量、超时时间等

### 4.2 入库方式规范

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 仅 JSON 入库 | 占用空间小、检索速度快 | 无完整判决书 | 快速入库、空间有限 |
| JSON + txt 文件入库 | 保留完整判决书 | 占用空间大 | 需要完整文书存档 |

### 4.3 质量检查清单

报告交付前逐项检查：

- [ ] 案情要素提取完整，无遗漏关键字段
- [ ] 罪名发现覆盖所有可能罪名，竞合关系已分析
- [ ] 罪名分析包含构成要件分析、本案符合性论证、竞合关系辨析
- [ ] 统计分析按用户确认的罪名进行过滤
- [ ] 典型类案覆盖量刑区间两端 + 中位数 + 指导案例
- [ ] 典型类案评析包含多维相似度比对表（含权重评分）
- [ ] 典型类案评析包含参考价值说明（星级评定）

## Tips

- **法释〔2011〕19号是核心参考**：危害计算机信息系统安全刑事案件的司法解释，是量刑情节认定的主要依据。
- **组合行为是常态**：计算机犯罪案件中多数为侵入+获取、获取+控制等组合，罪名发现时务必判断竞合关系。
- **§285-1 的系统范围是关键争点**：侵入行为指向的系统是否属于"国家事务、国防建设、尖端科学技术领域"，直接决定适用§285-1还是§285-2。
- **数据数量与量刑强相关**：获取数据的条数/金额是量刑档次划分的核心指标。
- **缓刑率是辩护重点**：计算机犯罪初犯、退赔退缴、认罪认罚的缓刑率较高。
