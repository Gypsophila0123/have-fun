## 一、四罪名构成要件详表

### 1. 非法侵入计算机信息系统罪（《刑法》第285条第1款）

| 要件 | 内容 |
|------|------|
| **客体** | 国家事务、国防建设、尖端科学技术领域的计算机信息系统的安全 |
| **客观方面** | 违反国家规定，侵入国家事务、国防建设、尖端科学技术领域的计算机信息系统 |
| **主体** | 一般主体（已满16周岁的自然人） |
| **主观方面** | 故意（明知是上述领域的计算机信息系统而非法侵入） |
| **行为性质** | 行为犯，无需造成实际危害后果即构成犯罪 |
| **量刑** | 3年以下有期徒刑或者拘役 |
| **加重情节** | 无加重档次 |

**关键认定要点**：
- "国家事务"领域：各级国家机关的政务信息系统
- "国防建设"领域：军队、国防科研单位的信息系统
- "尖端科学技术"领域：国家级科研院所的核心信息系统
- 侵入手段包括但不限于：破解密码、绕过认证、利用漏洞、使用 stolen credentials

### 2. 非法获取计算机信息系统数据、非法控制计算机信息系统罪（第285条第2款）

| 要件 | 内容 |
|------|------|
| **客体** | 第285条第1款规定以外的计算机信息系统的安全 |
| **客观方面** | 违反国家规定，获取该系统中存储、处理或者传输的数据，或者对该系统实施非法控制 |
| **主体** | 一般主体 |
| **主观方面** | 故意 |
| **行为性质** | 情节犯，需达到"情节严重"才构成犯罪 |
| **量刑（轻档）** | 情节严重：3年以下有期徒刑或者拘役，并处或者单处罚金 |
| **量刑（重档）** | 情节特别严重：3年以上7年以下有期徒刑，并处罚金 |

**关键认定要点**：
- "获取数据"：包括复制、下载、截取、查看等行为
- "非法控制"：包括获取控制权、植入后门、远程操控等
- 目标系统不能是第285条第1款的特定领域系统（否则构成侵入罪）
- 如果侵入第1款领域系统后获取数据，择一重罪论处

### 3. 提供侵入、非法控制计算机信息系统程序、工具罪（第285条第3款）

| 要件 | 内容 |
|------|------|
| **客体** | 计算机信息系统的安全秩序 |
| **客观方面** | 提供专门用于侵入、非法控制计算机信息系统的程序、工具，或者明知他人实施侵入、非法控制行为而为其提供程序、工具 |
| **主体** | 一般主体 |
| **主观方面** | 故意（明知程序、工具的用途） |
| **行为性质** | 情节犯，需达到"情节严重" |
| **量刑（轻档）** | 情节严重：3年以下有期徒刑或者拘役，并处或者单处罚金 |
| **量刑（重档）** | 情节特别严重：3年以上7年以下有期徒刑，并处罚金 |

**关键认定要点**：
- "专门用于"：程序、工具的设计目的即为侵入或控制，无合法用途
- "明知"：包括确切知道和应当知道
- 常见工具：木马程序、扫描工具、密码破解工具、DDoS工具
- 如果提供的工具具有合法用途（如渗透测试工具），需判断提供者的主观明知

### 4. 破坏计算机信息系统罪（第286条）

| 要件 | 内容 |
|------|------|
| **客体** | 计算机信息系统的功能、数据和程序的安全 |
| **客观方面** | 违反国家规定，对计算机信息系统功能进行删除、修改、增加、干扰，造成系统不能正常运行；或者对系统中存储、处理或者传输的数据和应用程序进行删除、修改、增加的操作；或者故意制作、传播计算机病毒等破坏性程序 |
| **主体** | 一般主体 |
| **主观方面** | 故意 |
| **行为性质** | 结果犯，需造成"后果严重" |
| **量刑（轻档）** | 后果严重：5年以下有期徒刑或者拘役 |
| **量刑（重档）** | 后果特别严重：5年以上有期徒刑 |

**关键认定要点**：
- 三种行为类型：破坏功能、破坏数据/程序、制作传播病毒
- "后果严重"：系统瘫痪、数据丢失、经济损失等量化标准见法释〔2011〕19号
- 与非法控制罪的区别：控制行为不必然导致系统不能正常运行，破坏行为则导致系统功能受损

## 二、罪名发现决策树

```
用户案情 → 行为类型判断
│
├─ 侵入行为
│  └─ 目标系统类型？
│     ├─ 国家事务 / 国防建设 / 尖端科技领域
│     │    → 候选罪名: 非法侵入计算机信息系统罪 (§285-1)
│     │    → 无需情节门槛，行为即构成犯罪
│     │
│     └─ 其他系统
│          → 候选罪名: 非法获取/控制罪 (§285-2)
│          → 侵入行为作为获取/控制行为的手段
│          → 需判断是否达到"情节严重"
│
├─ 获取数据行为
│  └─ 候选罪名: 非法获取计算机信息系统数据罪 (§285-2)
│     ├─ 目标系统为§285-1领域？ → 可能同时构成§285-1
│     ├─ 获取的是身份认证信息？ → 参考量化标准（10组/50组）
│     └─ 获取的是其他数据？ → 参考量化标准（10组/50组或经济损失）
│
├─ 非法控制行为
│  └─ 候选罪名: 非法控制计算机信息系统罪 (§285-2)
│     ├─ 目标系统为§285-1领域？ → 可能同时构成§285-1
│     └─ 控制台数 → 参考量化标准（10台/50台）
│
├─ 提供程序/工具行为
│  └─ 候选罪名: 提供侵入、非法控制计算机信息系统程序、工具罪 (§285-3)
│     ├─ 程序是否"专门用于"侵入/控制？
│     │  ├─ 是 → §285-3，参用量化标准（20人次/50人次）
│     │  └─ 否 → 可能不构罪，或构成实行犯的共犯
│     └─ 提供者是否"明知"用途？
│        ├─ 明知 → §285-3
│        └─ 不明知 → 不构罪
│
├─ 破坏行为
│  └─ 候选罪名: 破坏计算机信息系统罪 (§286)
│     ├─ 破坏对象类型？
│     │  ├─ 系统功能 → 删除/修改/增加/干扰功能
│     │  ├─ 数据/程序 → 删除/修改/增加数据和应用程序
│     │  └─ 病毒 → 故意制作、传播计算机病毒等破坏性程序
│     └─ 后果是否达到"严重"？
│        ├─ 是 → §286（5年以下）
│        └─ 特别严重 → §286（5年以上）
│
└─ 组合行为
   └─ 拆分后分别走上述分支
      ├─ 标注各行为的对应罪名
      └─ 判断竞合关系（见下表）
```

## 三、竞合关系判断规则

| 情形 | 处理方式 | 法律依据 |
|------|----------|----------|
| 侵入§285-1领域 + 获取数据 | 择一重罪论处（通常§285-2更重） | 想象竞合犯 |
| 侵入其他系统 + 获取数据 | §285-2（侵入为手段，获取为目的） | 牵连犯 |
| 侵入其他系统 + 非法控制 | §285-2（侵入为手段，控制为目的） | 牵连犯 |
| 侵入§285-1领域 + 破坏 | 分别评价，可能数罪并罚 | 数罪 |
| 非法控制 + 破坏（独立行为） | 分别评价，可能数罪并罚 | 数罪 |
| 非法控制 + 破坏（同一行为） | 择一重罪论处 | 想象竞合犯 |
| 提供工具 + 实行行为 | 提供者构成实行犯的共犯，择一重 | 共犯关系 |
| 获取数据 + 提供工具 | 可能数罪并罚 | 数罪 |
| 侵入 + 破坏同一系统功能 | §286（侵入为手段，破坏为目的） | 牵连犯 |

**竞合判断流程**：
1. 识别案情中的所有独立行为
2. 为每个行为匹配候选罪名
3. 判断行为之间的关系（手段-目的=牵连犯 / 同一行为触犯多罪名=想象竞合 / 不同行为触犯不同罪名=数罪）
4. 按对应规则处理
5. 在要件对比分析表中列出所有候选罪名及其竞合关系
